Interface EmbeddingRequest

Represents a request to generate embeddings. This structure encapsulates the text(s) to be embedded and any parameters that might influence the embedding process, such as model selection hints or user context.

EmbeddingRequest

interface EmbeddingRequest {
    texts: string | string[];
    modelId?: string;
    providerId?: string;
    userId?: string;
    collectionId?: string;
    customParameters?: Record<string, any>;
}

texts

texts: string | string[]

The text content to be embedded. Can be a single string or an array of strings for batch processing.

Example

// Single text
const requestOne: EmbeddingRequest = { texts: "Hello, world!" };
// Batch of texts
const requestBatch: EmbeddingRequest = { texts: ["First document.", "Second document."] };

Optional providerId

providerId?: string

Optional: The explicit ID of the LLM provider to use. This is typically used in conjunction with modelId. If modelId is provided and has a configured provider, this field might be used for validation or override if the architecture supports it. Generally, the model's configured provider is preferred.
